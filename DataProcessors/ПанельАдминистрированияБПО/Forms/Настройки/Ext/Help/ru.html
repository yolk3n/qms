· Рабочие места - Рабочие места с подключенным оборудованием.

· Рабочее место текущего сеанса - выбор рабочего места, подключение и настройка оборудования которого, производились ранее.

Поддержка ККТ с передачей данных в соответствии с действующей редакцией закона о применении контрольно-кассовой техники (54-ФЗ)

В соответствии с действующей редакцией Федерального закона 54-ФЗ, все организации и ИП, ведущие расчеты с покупателем с помощью наличных денежных средств или платежной карты обязаны перейти к применению контрольно-кассовой техники с передачей данных в ФНС через оператора фискальных данных (ОФД). Подробно о новом законодательстве можно ознакомиться в информационной системе 1С:ИТС.

Для поддержки нового порядка работы необходимо:

· модернизировать или обновить свой парк ККТ;

· поставить на учет в ФНС новые ККТ, в том числе и модернизированные;

· выбрать оператора фискальных данных (ОФД) и выполнить подключение новых касс.

Список моделей ККТ с передачей данных, которые можно подключить к программе размещен на странице http://v8.1c.ru/libraries/cel/certified.htm. По мере сертификации новых касс в этот перечень включаются новые модели.

Для подключения касс к оператору фискальных данных (ОФД) можно воспользоваться сервисом 1С-ОФД - https://portal.1c.ru/applications/56.

Подключение ККТ

Для оформления операций в розничной торговле к рабочему месту кассира должен быть подключен минимум один экземпляр ККТ. Ниже описано подключение и настройка ККТ с передачей данных в ОФД двух видов: АТОЛ и К1-Ф.

Подключение ККТ АТОЛ

Установка основной поставки драйвера

С сайта компании АТОЛ необходимо скачать дистрибутив драйвера торгового оборудования (далее ДТО) версии 10.5 или выше, который расположен в центре загрузок по следующему пути: Программное обеспечение – ДТО – 10.x.

В загруженном архивном файле необходимо открыть папку installer и из вложенной в нее папки exe запустить подходящий из файлов:

· KKT10-10.5.0.0-windows32-setup.exe – для 32-разрядного клиента 1С;

· KKT10-10.5.0.0-windows64-setup.exe – для 64-разрядного клиента 1С

Далее нужно подключить ККТ АТОЛ к ПК, по умолчанию установлен канал обмена через USB. Необходимо в Диспетчере устройств открыть раздел Порты (COM и LPT) и убедиться в наличии двух виртуальных портов, созданных при подключении.

Один виртуальный COM-порт – для подключения ККТ, второй – для службы передачи данных EoU.

Примечание: если СОМ-порты не обнаружены, но появились два Неизвестных устройства, то для них необходимо установить драйвер. Для этого: открыть карточку неизвестного устройства, на закладке Драйвер нажать кнопку Обновить драйвер и указать путь вручную. Путь к драйверу по умолчанию: C:\Program Files (x86)\ATOL\Drivers10\KKT\USB_Drivers. После установки драйвера в наименовании устройств появится отметка ATOL.

ККТ АТОЛ может быть подключена к ПК через один из портов:

· USB – рекомендуется выбирать при подключении одной ККТ;

· COM/VCOM – рекомендуется выбирать при подключении двух и более ККТ.

При выборе подключения через USB необходимо открыть драйвер и на вкладке Свойства выбрать из выпадающего списка модель подключенной ККТ, канал связи USB и нажать кнопку Проверка связи.

При выборе подключения через COM/VCOM необходимо открыть драйвер и на вкладке Свойства дополнительно нужно указать СОМ-порт, который был присвоен устройству:

· com-порт atol-usbcom_proxy1 - используется для подключения ККТ;

· com-порт atol-usbcom_proxy2 – используется для службы передачи данных EthernetOverUSB (далее EoU).

Далее необходимо выбрать модель подключенной ККТ, канал связи COM/VCOM и значение скорости 115200. После нажатия кнопки Проверка связи в поле Результат отобразятся сведения о модели ККТ.

Служба для работы через USB-кабель (EoU) устанавливается вместе с драйвером торгового оборудования. Нужно проверить определился ли порт для EoU в файле с настройками, который располагается по следующему пути:C:\ProgramData\ATOL\EoU в settings.xml. Если этого не произошло, то нужно прописать номер необходимого порта и сохранить файл. В примере для EoU был создан порт ATOL USB (COM5), поэтому должно быть указано значение 5.

Примечание: папка ProgramData по умолчанию имеет атрибут скрытый, поэтому предварительно необходимо открыть скрытые файлы и папки.

Далее необходимо запустить тест драйвера ККТ (Пуск – АТОЛ – Тест драйвера ККТ) и в режиме Свойства задать параметры в разделе 15 ОФД:

· адрес ОФД – demo.ofd.astralnalog.ru;

· порт ОФД – 7777.

· канал обмена с ОФД - USB (EoU).

Чтобы сохранить заданные параметры нужно нажать кнопку Применить.

Чтобы проверить связь с ОФД в тесте драйвера ККТ в разделе Нижний уровень нужно установить флажок Включено, ввести значение 82 01 06 00 в поле Буфер с командой ККТ и нажать кнопку Выполнить команду.

Подключение ККТ к компьютеру будет завершено, когда ККТ выдаст чек диагностики.

Подключение ККТ АТОЛ в «1С:Розница»

В «1С:Розница» в разделе Настройки подключаемого оборудования (меню Администрирование – Подключаемое оборудование) необходимо установить флажок Использовать подключаемое оборудование.

Затем необходимо нажать ссылку Подключаемое оборудование и в открывшейся форме Подключение и настройка оборудования выбрать тип оборудования ККТ с передачей данных. Далее нужно создать новый экземпляр оборудования (нажатием кнопки Создать) и заполнить следующие реквизиты:

· устройство используется – должен быть установлен флажок;

· тип оборудования - должно быть указано значение ККТ с передачей данных;

· драйвер оборудования – выбрать из списка актуальный обработчик драйвера для выбранной модели ККТ, например: АТОЛ:ККТ с передачей данных в ОФД (54-ФЗ) 10.Х;

· организация – указать организацию, от имени которой будут производиться операции на данной ККТ;

· рабочее место – будет заполнено автоматически данными из системы;

· наименование – сформируется автоматически на основании введенных реквизитов;

· серийный номер ККТ – ввести с клавиатуры заводской номер, указанный на корпусе устройства.

После сохранения элемента на данной форме станут доступны команды: настроить, операции с ФН и параметры ККТ.

Для настройки подключения к ККТ необходимо нажать кнопку Настроить и в открывшемся окне заполнить реквизиты: модель и порт (USB).

Чтобы проверить подключение нужно нажать кнопку Тест устройства. При корректной настройке появится сообщение об успешно выполненном тесте с указанием информации о подключенной ККТ.

Подключение ККТ К1-Ф

Предварительная настройка ККТ

Первоначально необходимо подключить ККТ к локальной сети и сети электропитания. Далее нужно нажать кнопку питания, расположенную на задней стенке ККТ и зажать кнопку промотки ленты (FEED). Распечатается чек тестирования со следующей информацией: текущие дата и время, интерфейс подключения по сети, МАС адрес, протокол работы, IP адрес, маска сети, шлюз, DNS, версия прошивки (в формате год-месяц-день буква Т и время), заводской номер кассы и регистрационный номер (если касса уже зарегистрирована).

Затем в любом браузере необходимо перейти по IP адресу ККТ, который указан на тестовом чеке.

Откроется окно входа в меню настроек ККТ, в котором нужно ввести учетные данные:

· логин:admin;

· пароль: 12.

В разделе Настройки нужно выбрать пункт ОФД и указать следующие настройки доступа:

· URL – demo.ofd.astralnalog.ru;

· Порт – 7777.

Введенные данные нужно сохранить, нажав кнопку Сохранить.

Примечание: в верхнем правом углу должны отображаться актуальные дата и время. В противном случае необходимо обновить браузер или отключить антивирусное ПО.

В разделе Настройки нужно выбрать пункт ККТ, в Параметрах установить флажок Разрешить работу с маркированным товаром и сохранить данные.

Подключение ККТ К1-Ф в «1С:Розница»

В «1С:Розница» в разделе Настройки подключаемого оборудования (меню Администрирование – Подключаемое оборудование) необходимо установить флажок Использовать подключаемое оборудование.

Затем необходимо нажать ссылку Подключаемое оборудование и в открывшейся форме Подключение и настройка оборудования выбрать тип оборудования ККТ с передачей данных. Далее нужно создать новый экземпляр оборудования (нажатием кнопки Создать) и заполнить следующие реквизиты:

· устройство используется – установить флажок;

· тип оборудования - должно быть указано значение ККТ с передачей данных;

· драйвер оборудования – выбрать из списка актуальный обработчик драйвера для выбранной модели ККТ, например: Версия-Т:ККТ с передачей данных в ОФД (54-ФЗ);

· организация – указать организацию, от имени которой будут производиться операции на данной ККТ;

· рабочее место – будет заполнено автоматически данными из системы;

· наименование – сформируется автоматически на основании введенных реквизитов;

· серийный номер ККТ – ввести с клавиатуры заводской номер, указанный на корпусе устройства.

После сохранения элемента на данной форме станут доступны команды: настроить, операции с ФН и параметры ККТ.

Для настройки подключения к ККТ необходимо нажать кнопку Настроить и в открывшемся окне заполнить реквизиты:

· адрес ККТ – ввести IP адрес, напечатанный в тестовом чеке (по умолчанию имеет значение 192.168.1.101);

· TCP порт - должен иметь значение 49 152;

· таймаут (сек.) – можно оставить значение по умолчанию (5). Если касса не будет успевать получать данные в течение этого времени, то необходимо увеличить время;

· пароль на связь - ast-01;

· логин пользователя ККТ - admin;

· пароль пользователя ККТ - 12.

Чтобы проверить подключение нужно нажать кнопку Тест устройства. При корректной настройке появится сообщение об успешно выполненном тесте с указанием информации о подключенной ККТ.

 

Пользователю необходимо выбрать тип оборудования ККТ с передачей данных и создать новый элемент справочника. Далее необходимо выбрать организацию, от имени которой будет производиться продажа товаров на данном фискальном регистраторе и драйвер устройства. В поле серийный номер вводится заводской номер, казанный на корпусе устройства. Затем необходимо нажать кнопку Настроить и выполнить необходимые настройки драйвера - порт, сетевые настройки и другие настройки, характерные для данной модели. Нажать Тест устройства.

При успешном подключении устройства на экране появится сообщение. Пользователь может закрыть форму с помощью кнопки Записать и закрыть.

Регистрация фискального накопителя

После подключения ККТ к программе необходимо зарегистрировать фискальный накопитель. Эта операция производится также при замене фискального накопителя после окончания срока действия или исчерпании ресурса памяти: использованный фискальный накопитель закрывается, затем производится замена физического устройства и регистрируется новый фискальный накопитель.

Для проведения операции необходимо выбрать пункт Регистрация из меню Операции с фискальным накопителем. При этом откроется форма, на которой необходимо заполнить поле Регистрационный номер ККТ номером, полученным при регистрации кассы в ФНС, проверить правильность заполнение реквизитов Организации, указать Адрес установки ККТ.

Пользователь может выбрать одну или несколько систем налогообложения, если на данном аппарате планируется выбивать чеки для регистрации расчетов по товарам и услугам, относящимся к видам деятельности с различными формами налогообложения. При этом, в случае если продажи оформляются в РМК программа контролирует состав чека: в один чек могут входить товары, остатки которых списываются со складов с одной и той же системой налогообложения. Списание товаров производится в соответствии с настройкой распределения продаж по складам и услуг по организациям. Подробнее см. "Описание конфигурации Розница, Ведение информации о магазинах.

Настройки ККТ включают в себя ряд признаков ведения деятельности. Пользователю также необходимо указать наименование и ИНН оператора фискальных данных.

Регистрационные данные ККТ доступны к просмотру в карточке экземпляра подключаемого оборудования.

Операция Изменение параметров регистрации необходима при перерегистрации фискального накопителя. В зависимости от выбранной причины перерегистрации пользователю доступны для изменения те реквизиты ККТ, которые он может изменить.

Зарегистрированное устройство может быть выбрано в карточке Кассы ККМ. Подробнее см. "Описание конфигурации Розница, редакция 2.2", раздел 2.2.5 Ведение информации о кассах предприятия.

Важно! Обращаем внимание пользователей, что распределение выручки по секциям ФР для типа оборудования ККТ с передачей данных не поддерживается. В настройках Касс ККМ для типа оборудования ККТ с передачей данных в поле Распределение выручки по секциям ФР подставляется предустановленный элемент справочника настроек, который нет возможности изменить. Работа с фискальными регистраторами поддерживается в прежнем объеме.

Подключение сканера штрихкода 2D

В «1С:Розница» в разделе Настройки подключаемого оборудования (меню Администрирование – Подключаемое оборудование) должен быть установлен флажок Использовать подключаемое оборудование.

Затем необходимо нажать ссылку Подключаемое оборудование и в открывшейся форме Подключение и настройка оборудования выбрать тип оборудования Сканеры штрихкода. Далее нужно создать новый экземпляр оборудования (нажатием кнопки Создать) и заполнить/проверить следующие реквизиты:

· устройство используется – должен быть установлен флажок;

· тип оборудования - должно быть указано значение Сканеры штрихкода;

· драйвер оборудования – выбрать из списка актуальный драйвер для выбранной модели сканера, например: 1С:Сканеры штрихкода (NativeApi) 1111 или 1С:Сканеры штрихкода (NativeApi);

· рабочее место, наименование – будут заполнены автоматически данными из системы;

· серийный номер – необязательный реквизит. Можно ввести с клавиатуры заводской номер, указанный на корпусе устройства.

После сохранения элемента необходимо нажать появившуюся на форме кнопку Настроить.

В открывшейся форме все реквизиты будут заполнены автоматически. Чтобы проверить корректность настройки необходимо нажать кнопку Тест подключения и отсканировать Data matrix код с любой пачки или блока. В открывшемся окне Тест подключения появится строчка с кодом товара и суффиксом CR в конце. После закрытия данной формы программа проинформирует об успешно выполненном тесте.

Примечание: если в строчке отразились названия клавиш, то необходимо для реквизита Символ GS изменить значение с NONE на клавишу, которой сканером будет передаваться символ GS (#29).

Настройка НСИ

Для маркируемой продукции рекомендуется создавать новый вид номенклатуры и новую карточку номенклатуры с уникальным GTIN, а не править существующие немаркируемые элементы.

Для видов номенклатур товаров, подлежащих обязательной маркировке, необходимо установить особенность учета табачной продукции. Для этого в открытой карточке вида номенклатуры нужно нажать ссылку Изменить, расположенную напротив реквизита Тип номенклатуры, и указать значение Маркируемая табачная продукция. После этого признак предмета расчета примет значение Подакцизный товар.

Примечание: тип номенклатуры является ключевым реквизитом, поэтому предварительно необходимо разрешить его редактирование. Для этого в открытой карточке вида номенклатуры необходимо нажать кнопку Еще, выбрать значение Разрешить редактирование реквизитов и подтвердить намерения об изменении, отвечая на уточняющие вопросы программы.
