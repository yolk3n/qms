Обработка "Свертка информационной базы"

С помощью данной обработки можно выполнить свертку информационной базы. Для выполнения свертки потребуется указать дату свертки и выбрать режим выполнения свертки. В результате свертки информационной базы будут удалены документы до даты свертки включительно, и сформированы входящие остатки.

Внимание. Операция свертки информационной базы является необратимой. Перед сверткой необходимо создать копию информационной базы

Общие сведения

Дата свертки информационной базы

В качестве даты свертки указывается последняя дата, входящая в сворачиваемый период. Например, при свертке данных на начало 2018 года необходимо указать дату свертки 31.12.2017.

Выбирать дату свертки необходимо с учетом следующих факторов:
  • Дата свертки должна соответствовать окончанию периода сдачи регламентированной отчетности (конец квартала, конец года)
  • В сворачиваемом периоде должны быть завершены все операции: период закрыт, регламентированная отчетность сдана

Ограничения механизма свертки информационной базы

Некоторые данные не могут быть свернуты:
  • Товары в резерве на складах (документы в статусе "К отгрузке", "В работе" и т.д.) 
  • Незавершенные отгрузки товаров
  • Неоформленные излишки и недостачи 

Эти сведения при свертке будут удалены. Не рекомендуется выполнять свертку информационной базы, если существуют указанные ограничения.

Режим выполнения свертки

Перед выполнением свертки информационной базы потребуется выбрать режим ее выполнения:
  • Автоматический режим. Все этапы свертки будут выполняться последовательно за один прием.
  • Контролируемый режим. Каждый этап свертки будет выполняться обособленно. В этом режиме можно остановить процесс свертки после выполнения одного из этапов, и закрыть форму помощника. При следующем открытии формы процесс будет продолжен с того же места.

Этапы свертки информационной базы

  • Формирование документов ввода остатков
  • Удаление движений регистров, пометка на удаление документов
  • Проведение документов ввода остатков

Выполнение свертки информационной базы

Действия, которые необходимо выполнить перед сверткой

В сворачиваемом периоде должны быть выполнены:
  • Синхронизация данных (если она используется) 
  • Отражены все излишки и недостачи
  • Оприходованы в кассы все денежные средства  
  • Завершены отпуски товаров в отделения и передачи товаров 
  • Выполнено архивирование и удаление чеков ККМ 
  • Выполнены все операции, связанные с закрытием месяца

В форме помощника свертки информационной базы предусмотрен отчет, который показывает:

  • Под какие ограничения попадает информационная база
  • Какие операции не выполнены на дату свертки 
Отчет доступен по гиперссылке после выбора даты свертки, на следующей странице помощника. Если перечисленных ограничений и невыполненных операций нет, гиперссылка на форме не отображается. 

Последовательность действий при свертке информационной базы

  • Создать резервную копию информационной базы
  • Указать дату свертки
  • Выбрать режим выполнения свертки (автоматический либо контролируемый)
  • Запустить процесс свертки, дождаться его окончания
  • Проверить результаты свертки (с помощью прикладных отчетов) 
  • Выполнить удаление данных в свернутом периоде (с помощью специальных обработок) 
  • Закрыть свернутый период для изменений: установить дату запрета изменения данных

Если свертка информационной базы выполняется впервые, рекомендуется провести ее на копии информационной базы. Если свертка пройдет успешно, процесс следует повторить на рабочей базе. 

Свертка должна выполняться от имени пользователя, обладающего правами администратора.

Если при выполнении свертки возникли проблемы, и процесс свертки прервался, необходимо повторно открыть форму помощника свертки. В открывшейся форме будет отображена информация: можно ли продолжить свертку дальше, либо необходимо восстановить информационную базу из резервной копии.

Проверка результатов свертки информационной базы

Информация о ходе свертки фиксируется в журнале регистрации. Его можно открыть с помощью соответствующей гиперссылки, которая доступна после окончания процесса свертки.

Для проверки учетных данных можно использовать отчеты, например:

  • Себестоимость товаров 
  • Остатки товаров на складах, Остатки товаров на складах в отделениях

Удаление данных после свертки

После того, как выполнена свертка информационной базы, и проверен ее результат, необходимо выполнить удаление данных за свернутый период. Для этого следует использовать обработку "Удаление помеченных объектов". К ней можно перейти с помощью специальной гиперссылки, которая появляется после свертки. Порядок работы с ней приведен во встроенной справке. 

Особенности свертки в условиях использования РИБ

Если используется распределенная информационная база, при выполнении свертки должны соблюдаться следующие правила
  • Перед сверткой должны быть выполнены все обмены
  • Свертка должна выполняться в главном узле
  • После выполнения свертки следует выполнить обмен данными в распределенной базе
  • Перед выполнением обмена рекомендуется в главном узле выполнить удаление помеченных на удаление объектов

Особенности свертки некоторых учетных данных

Товары

Остатки товаров (собственные, собственные в отделениях) вводятся с помощью документа "Ввод остатков" с соответствующим видом операции. Остатки регистрируются в разрезе организаций и складов. Остатки собственных товаров, для которых известен поставщик, вводятся с указанием поставщика. Подразделения и отделения определяются по основному складу (если склад является основным для одного подразделения или отделения).

Остатки товаров, размещенные по местам хранения, вводятся с помощью документов "Размещение товаров по местам хранения" и "Размещение товаров по местам хранения в отделениях".

При необходимости остатки формируются в разрезе серий, партий и источников финансирования.

Незаконченные операции

Остатки по незаконченным операциям формируются соответствующими документами:
  • Спецификации к договору 
  • Заказ поставщику 
  • Заказ на перемещение
  • Требование отделения 
  • Назначение врача

Если операции на дату свертки еще не начали выполняться, сохраняются исходные документы. Если операции частично выполнены, исходные документы изменяются:

  • исполненная часть удаляется
  • неисполненная часть остается

Прочее

Цены номенклатуры устанавливаются по срезу последних на дату свертки.
