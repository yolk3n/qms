Требование отделения

Документ Требование отделения служит для отражения текущих потребностей отделений медицинского учреждения в аптечных товарах. Требование является электронным заказом, адресованным на аптечный склад, на поступление необходимых препаратов в отделение. Требование-накладная на лекарства выписывается персоналом отделения медицинского учреждения.

Для документа Требование отделения можно установить приоритет, в зависимости от важности задачи по скорейшему выполнению требования.

При необходимости разделения требований отделений по группам предметно-количественного учета в документе указывается значение реквизита Группа ПКУ. По указанному значению осуществляется отбор в форме подбора товаров в табличную часть документа Требование отделения на закладке Товары.

Для планирования поступления лекарственных средств в отделения используется механизм заполнения желаемой даты поступления. По умолчанию планируемая дата поступления для всех строк табличной части документа одинакова, и на закладке Товары установлен соответствующий флажок, тогда, если надо, можно заполнить дату поступления. Если по разным позициям требования предполагаются разные сроки поступления товаров, то флажок надо снять, при этом в табличной части документа у каждой строки появится графа Дата поступления. Для упрощения заполнения даты поступления в строках табличной части можно воспользоваться кнопкой Заполнить - Дата поступления для выделенных позиций товаров.

По каждой строке табличной части документа Требование отделения существует возможность ввести пациентов, на которых будут расходоваться затребованные медикаменты. Строки, по которым введены данные по пациентам, помечаются соответствующей пиктограммой в левой колонке табличной части документа.

Для документа Требование отделения в программе существует возможность управлять статусами документа путем прохождения электронного документа по цепочке взаимосвязанных задач согласования и исполнения. Возможность использования в программе бизнес-процессов по документам Требование отделения контролируется опцией Согласование и исполнение требований отделений в настройках параметров учета раздела Администрирование-Отделения. Для документа реализованы бизнес-процессы "Согласование" и "Исполнение", а так же составной бизнес-процесс "Согласование и Исполнение".
Единовременно по одному документу может быть сформирован только один бизнес-процесс.
Если флажок согласования и исполнения требований отделений не установлен, то статусы требования меняются вручную.

Статус документа информирует о том, в каком состоянии находится сейчас бизнес-процесс по документу. При задании роли пользователю Разрешить изменение статуса требований отделений в ручную пользователю разрешается менять статус документа, иначе значения статуса изменяются только программно, на основании этапов бизнес-процесса.
Документ может находиться в одном из 6 предопределенных статусов:

  • Не согласован - устанавливается при создании документа Требование отделения
  • На согласовании - по документу Требование отделения запущен бизнес-процесс Согласование
  • Согласован - по документу Требование отделения завершены задачи в рамках бизнес-процесса Согласование
  • К выполнению - по документу Требование отделения запущен бизнес-процесс Исполнение
  • Исполнен - по документу Требование отделения завершены задачи в рамках бизнес-процесса Исполнение
  • Закрыт - по документу Требование отделения завершены задачи по ознакомлению с результатами бизнес-процесса Исполнение

По документу Требование отделения можно анализировать состояния выполнения требования с помощью отчета "Состояния выполнения требований отделений".

При проведении документа по регистру накопления формируются соответствующие записи отражающие в программе факт оформления требования отделения к складу аптеки .

Документ служит для формирования типовой межотраслевой печатной формы 0504204.
