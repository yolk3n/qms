Номенклатура

В справочнике Номенклатура хранится полный перечень аптечных товаров, используемых в текущей деятельности больничной аптеки. Справочник используется для ведения списка номенклатуры любых товарно-материальных ценностей, учитываемых на складе больничной аптеки: лекарственных средств (готовых, субстанций, изготавливаемых), перевязочных средств, биологически активных добавок, расходных материалов для медицинского оборудования, кровекомпонентов, изделий медицинского назначения т.д.

Справочник является многоуровневым, т.е. в нем можно объединять номенклатуру в произвольные группы и подгруппы.

Элементы справочника Номенклатура рекомендуется загружать из сервиса 1С:Номенклатура.
В этом случае все реквизиты элемента номенклатуры заполняются автоматически на основании данных сервиса.

Важно! Загрузка лекарственных средств из базы данных Регистра лекарственных средств (с диска ИТС:МЕДИЦИНА или сайта ИТС аптечных товаров) не поддерживается.

Сервис 1С:Номенклатура – это единый каталог описаний товаров и услуг, используемых в конфигурациях на платформе "1С:Предприятии 8". Карточка товара содержит наиболее часто требуемые свойства товара, необходимые для заведения номенклатурной позиции в учетных системах, такие как: наименование, ставка НДС, единица измерения и прочие.

Интеграция с сервисом дает пользователям следующие возможности:
  • Загрузка новой номенклатуры
  • Сопоставление номенклатуры, если номенклатура уже была создана ранее, например, при загрузке данных из базы данных РЛС

Загрузка новой номенклатуры


Загрузка номенклатура из сервиса возможна несколькими способами:
  • из формы списка справочника Номенклатура (подменю 1С:Номенклатура - Загрузить номенклатуру);
  • из формы элемента справочника Номенклатура (реквизит 1С:Номенклатура).

Важно! Загрузка лекарственных средств выполняется из категории "Единый справочник–каталог лекарственных препаратов (ЕСКЛП)" (группа категорий Медицина).
Особенностью данной категории является то, что данные номенклатуры этой категории имеют более широкий набор свойств, характеризующий лекарственное средство, и позволяющий сформировать данные "Аптечного товара".

Сопоставление номенклатуры


Сопоставление номенклатуры можно выполнить несколькими способами:
  • из формы списка справочника Номенклатура (подменю 1С:Номенклатура - Сопоставить выделенные или Сопоставить все);
  • из формы списка справочника Номенклатура при нажатии на гиперссылку в поле 1С:Номенклатура.
При сопоставлении номенклатуры с сервисом 1С:Номенклатура есть возможность перезаполнить сопоставленную номенклатуру, а так же включить автоматическое обновление.
При этом, автоматическое перезаполнение и обновление выполняется только для реквизитов, которые не включены в состав блокируемых реквизитов, изменение которых влияет на учет.
Для обновление таких реквизитов необходимо выполнить ручное обновление из карточки справочника Номенклатура (при нажатии на картинку рядом с полем 1С:Номенклатура, выбрать Ручное обновление).

Ручное создание элемента справочника


При создании нового элемента справочника вручную программа предлагает заполнить реквизиты, в которых будут храниться сведения о номенклатурной единице. В шапке элемента надо указать:
  • Рабочее наименование – указывается краткое наименование номенклатуры, используется для выбора из справочника и печати в отчетных и первичных документах.
  • Наименование для печати– наименование позиции номенклатуры, которое будет печататься во всех документах. В первичных документах будет печататься полное наименование, если оно задано, если нет, то краткое наименование.
  • Артикул - артикул позиции справочника номенклатуры (если это необходимо).
  • Код - уникальный код позиции справочника номенклатуры (генерируется программой автоматически).

В группе Основные параметры учета необходимо выбрать вид номенклатуры, к которому будет относиться новый товар. Этот реквизит устанавливает основные параметры, определяющие порядок аналитического учета по номенклатурной позиции. В этой же группе надо ввести срок годности для товаров, являющихся лекарственным средством. При установке флажка Вести учет по ГТД в документах поступления можно будет указать номер ГТД товара.

При выборе вида номенклатуры с типом номенклатуры Лекарственное средство элемент справочника Номенклатура надо связать с элементом справочника Аптечные товары путем заполнения реквизита Аптечный товар в группе Данные препарата соответствующим элементом. Для каждого элемента справочника Аптечные товары может быть создан один элемент справочника Номенклатура. При этом в группе Данные препарата появится информация об аптечном товаре.

В группе Классификация надо указать, в какую группу справочника будет входить новый элемент, и к какой группе ПКУ он будет относиться. Для формирования отчета АВС/VEN - анализ, то следует заполнить реквизит VEN классификация. Если в организации ведется учет розничных продаж, то надо заполнить реквизит Ценовая группа. Это дает возможность установить разные наценки одному виду цены, в зависимости от того, для какой ценовой группы товаров рассчитывается цена.

В группе Общероссийские классификаторы указываются значения из общероссийских классификаторов.

В группе Описание можно выбрать изображение номенклатуры, а так же в поле Текстовое описание ввести любой текст, который пользователь считает нужным.

Единицы измерения


Каждая позиция номенклатуры характеризуется базовой единицей измерения. В этой единице измерения хранятся остатки товаров в регистрах. Базовая единица измерения задается путем выбора значения из списка единиц измерения. Для номенклатуры лекарственных средств и биологически активных добавок, связанных со справочником аптечных товаров, базовая единица измерения автоматически заполняется соответствующим реквизитом элемента справочника «Аптечные товары». по следующему алгоритму: фактически, из всех возможных единиц измерения аптечного товара выбирается минимальная.
Основная единица учета - если этот реквизит заполнен, то данная единица измерения будет автоматически подставляться в документы. Количество в отчетах о движениях и остатках товаров будет отражаться в основной единице учета.
Минимальная единица отпуска из аптеки - единица, меньше которой отпуск из аптеки не производится. Если в требовании отделения указать единицу меньше минимальной, то при автоматическом расчете отпускаемого количества номенклатуры со склада округление всегда идет в большую сторону до целого количества в минимальных единицах отпуска из аптеки (например, если минимальной единицей является блистер по 10 таблеток, а в требовании указано 15 таблеток, то при автоматическом подборе в документ программа установит количество товара в размере двух блистеров). Если этот реквизит не заполнен, то в документ подставляется основная единица учета. В документе, если это необходимо, единицы измерения товара можно изменять.

Важно! Для номенклатуры лекарственных средств и биологически активных добавок, минимальная единица измерения которой поддается штучному учету (например, таблетки), базовая единица не доступна для изменения. Для номенклатуры лекарственных средств и биологически активных добавок, минимальная единица измерения которой не поддается штучному учету и является единицей изменения веса или объема, базовая единица доступна для изменения.

По кнопке Перейти открывается список команд, по которым можно ввести параметры, дополнительно характеризующие данную позицию номенклатуры:

По команде Упаковки - можно увидеть упаковки для данной номенклатуры, если элемент создан на основании аптечного товара. Если элемент справочника создается вручную или требуется ввести для созданного на основании аптечного товара более крупную упаковку, то по кнопке Создать необходимо заполнить реквизиты, характеризующие данную упаковку:

  • Упаковка - выбрать из списка вид упаковки;
  • Флажок Первичная упаковка устанавливается для первой создаваемой вручную упаковки;
  • Флажок Состоит из других упаковок устанавливается, если создается упаковка более крупная, чем созданная предварительно;
  • реквизит В этой упаковке - проставляется количество, показывающее, сколько в этой упаковке единиц измерения, указанных справа от реквизита;
  • Наименование и Полное наименование - можно автоматически сгенерировать по кнопке с пиктограммой .

По команде Лекарственные прописи можно ввести лекарственную пропись, если создаваемая номенклатура имеет тип экстемпоральной рецептуры.

По команде Серии номенклатуры отражается список серий, которые вводятся в документах при поступлении товара, в этом списке также можно создать новые серии номенклатуры.

По команде Штрихкоды можно ввести штрихкод как производителя так и внутренний штрихкод, если используется внутреннее штрихкодирование.

По команде Прикрепленные файлы - к элементу Номенклатура можно присоединить любые файлы, если это необходимо,

По команде Цены номенклатуры - можно посмотреть все цены товара, созданные документом Установка цен, если в медицинском учреждении ведется розничная продажа.
