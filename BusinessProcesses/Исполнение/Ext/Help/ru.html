Исполнение

Этот бизнес-процесс используется для автоматизации исполнения распоряжений руководителя одним или несколькими исполнителями с одновременным контролем.

Если указано несколько Исполнителей бизнес-процесса, то одного из них можно указать ответственным исполнителем. При старте этого бизнес-процесса соответствующие задачи на исполнение автоматически получат все исполнители по списку. Из задачи ответственного исполнителя имеется возможность просмотреть задачи остальных исполнителей.

Пользователь, указанный в поле Контролер, получит задачу контролировать одновременно с исполнителями. В обязанности контролера обычно входит слежение за сроками исполнения.

Пользователь, указанный в поле Проверяющий, получит задачу после того, как все исполнители выполнили свои задачи. В обязанности проверяющего входит проверка результата исполнения задач. Проверяющий может возвращать задачи исполнителей на доработку при помощи соответствующей команды в карточке своей задачи.

Для просмотра текущего состояния бизнес-процесса (списка выполненных и не выполненных задач) используйте команду Задачи в панели навигации карточки бизнес-процесса.

При старте этого бизнес-процесса документу автоматически будет установлено состояние На исполнении и при успешном завершении бизнес-процесса - состояние Исполнен, затем Закрыт.

Если бизнес-процесс стартован и по нему есть незавершенные задачи, то при изменении значений реквизитов в карточке бизнес-процесса будет выполнено изменение реквизитов незавершенных задач. Это позволяет оперативно вносить изменения в описания выполняемых задач.
При редактировании стартованного бизнес-процесса в карточке бизнес-процесса доступны для изменения только те поля, которые можно изменять после старта, и которые будут перенесены в соответствующие задачи.
