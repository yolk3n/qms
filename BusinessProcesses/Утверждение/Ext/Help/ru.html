Утверждение

Этот бизнес-процесс используется для передачи исходящего или внутреннего документа на утверждение ответственным лицам организации.

При старте бизнес-процесса руководителю поступает задача Утвердить, содержащая ссылку на документ. После того как ответственное лицо рассмотрит документ и отметит в задаче результат утверждения, автору поступает задача Ознакомиться с результатами утверждения. В случае отрицательного результата из этой задачи автор может отправить документ на повторное утверждение.

Этот бизнес-процесс можно создавать непосредственно из карточки, например, исходящего документа, при помощи кнопки Создать на основании - Процесс.

При выборе утверждающего можно указать не только конкретного пользователя, но и роль исполнителя.

Для просмотра текущего состояния бизнес-процесса (списка выполненных и не выполненных задач) используйте команду Задачи в панели навигации карточки бизнес-процесса.
При старте этого бизнес-процесса документу Формуляр автоматически будет установлено состояние На утверждении и при успешном завершении бизнес-процесса - состояние Утвержден.

Если бизнес-процесс стартован и по нему есть незавершенные задачи, то при изменении значений реквизитов в карточке бизнес-процесса будет выполнено изменение реквизитов незавершенных задач. Это позволяет оперативно вносить изменения в описания выполняемых задач.
При редактировании стартованного бизнес-процесса в карточке бизнес-процесса доступны для изменения только те поля, которые можно изменять после старта и которые будут перенесены в соответствующие задачи.

При установленной настройке программы Использовать электронно-цифровые подписи и шифрование) становится доступной флажок Подписывать ЭЦП, тогда при включенном флажке Подписывать ЭЦП ответственным лицом нужно будет установить электронно-цифровую подпись (ЭЦП).
